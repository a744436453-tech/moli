# Moli 项目目录整理方案

## 1. 目录目标

本仓库按「视觉稿 / 前端 Demo / 产品说明」三类内容管理，避免图片、代码和文档混在同一层级。

```text
moli/
├── 原型图/
│   ├── 01-首页陪伴/
│   ├── 02-数字形象生成/
│   ├── 03-Momo聊天/
│   ├── 04-回忆时间线/
│   ├── 05-互动玩法/
│   ├── 06-我的与设置/
│   └── 99-历史版本/
├── demo/
│   ├── index.html
│   ├── styles.css
│   └── app.js
└── docs/
    ├── 目录整理方案.md
    └── 功能板块说明.md
```

## 2. 视觉稿分板块规则

### 01-首页陪伴
用于存放首页、宠物主视觉、今日状态、快捷互动入口、首页空状态、首页加载态等视觉稿。

### 02-数字形象生成
用于存放上传照片、选择风格、补充性格、生成中、生成成功、生成失败、重新生成等视觉稿。

### 03-Momo聊天
用于存放聊天首页、文字消息、语音消息、回忆触发、快捷问题、长按菜单、情绪反馈等视觉稿。

### 04-回忆时间线
用于存放相册、时间线、纪念日、故事详情、新增回忆、编辑回忆等视觉稿。

### 05-互动玩法
用于存放抚摸、投喂、玩球、拍照、互动完成反馈、宠物状态变化等视觉稿。

### 06-我的与设置
用于存放宠物档案、素材库、隐私设置、数据导出、纪念册生成等视觉稿。

### 99-历史版本
用于存放废弃方案、早期探索、风格对比，不参与当前 Demo 主流程。

## 3. 命名规则

建议使用：

```text
板块编号-页面名称-状态-版本号.png
```

示例：

```text
01-首页陪伴-默认态-v1.png
02-数字形象生成-上传照片-v1.png
03-Momo聊天-回忆触发-v1.png
05-互动玩法-玩球反馈-v1.png
```

## 4. 当前 Demo 范围

当前 `demo/` 已覆盖 6 个功能板块：

1. 首页陪伴
2. 生成数字形象
3. Momo 聊天
4. 回忆时间线
5. 互动玩法
6. 我的与设置

Demo 采用纯 HTML / CSS / JS，可直接打开 `demo/index.html` 预览。
