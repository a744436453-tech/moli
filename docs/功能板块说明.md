# Moli 功能板块说明

## 1. 首页陪伴

首页不是功能导航页，而是用户每天进入后与宠物建立连接的第一屏。

展示重点：

1. 宠物数字形象作为页面主角，保持足够大面积展示。
2. 展示宠物今日状态，例如心情、想做的互动、最近回忆。
3. 快捷入口只保留高频动作：摸摸、投喂、玩球、聊天。
4. 避免会员、广告、复杂运营入口干扰情绪体验。

当前 Demo 已实现：

1. 宠物主视觉。
2. 今日状态卡片。
3. 快捷互动按钮。
4. 抚摸与聊天入口反馈。

## 2. 数字形象生成

数字形象生成是 MVP 的关键转化路径，需要让用户感觉简单、安全、可控。

核心流程：

1. 上传宠物照片。
2. 选择视觉风格。
3. 补充性格、习惯、叫声、重要记忆。
4. 生成数字形象。
5. 进入首页陪伴或继续完善资料。

当前 Demo 已实现：

1. 上传照片卡片。
2. 风格选择。
3. 性格输入。
4. 开始生成按钮反馈。

## 3. Momo 聊天

聊天模块要接近微信体验，降低学习成本，但内容反馈需要更温柔，避免机械问答。

核心能力：

1. 文字聊天。
2. 语音聊天。
3. 回忆触发。
4. 宠物口吻回复。
5. 快捷情绪安慰。

当前 Demo 已实现：

1. 聊天气泡。
2. 回忆触发提示。
3. 输入栏。
4. 发送反馈。

## 4. 回忆时间线

回忆模块承载照片、视频、故事、纪念日，是用户长期沉淀内容的核心板块。

核心能力：

1. 按时间展示宠物重要节点。
2. 支持新增照片、视频、文字故事。
3. 支持纪念日提醒。
4. 支持从聊天和互动中自动生成回忆卡片。

当前 Demo 已实现：

1. 时间线结构。
2. 多条回忆卡片。
3. 新增回忆入口。

## 5. 互动玩法

互动模块需要让用户感受到“宠物有回应”，而不是只点击按钮。

核心能力：

1. 抚摸反馈。
2. 投喂反馈。
3. 玩球反馈。
4. 拍照保存。
5. 宠物心情变化。

当前 Demo 已实现：

1. 互动场景。
2. 玩球元素。
3. 四类互动按钮。
4. Toast 即时反馈。

## 6. 我的与设置

我的页面不作为主要转化页，MVP 阶段主要承载基础资料、素材管理和隐私控制。

核心能力：

1. 宠物档案管理。
2. 照片与素材库。
3. 隐私与数据导出。
4. 纪念册生成。

当前 Demo 已实现：

1. 用户基础信息。
2. 功能列表。
3. 设置入口结构。

## 7. 后续建议

下一步可以按真实视觉稿继续补充：

1. 将 `原型图/` 下的图片按板块目录归档。
2. 在 `demo/` 中增加真实图片引用。
3. 为每个板块增加二级页面，例如生成中、聊天语音态、回忆详情、互动成功态。
4. 根据实际技术栈改造成 React / Vue / Next.js 项目。
